# DPRR Data

Sharded RDF dataset from the [Digital Prosopography of the Roman Republic](https://romanrepublic.ac.uk) (DPRR).

Each person in the prosopography has their own Turtle file containing their full record — biographical data, office-holdings, family relationships, status assertions, scholarly notes, and source references.

## Layout

```
ontology.ttl              # OWL/RDFS schema definitions
reference/
  offices.ttl             # 204 political and religious offices
  provinces.ttl           # 92 Roman provinces
  praenomina.ttl          # 45 Roman first names
  tribes.ttl              # 37 voting tribes
  relationships.ttl       # 44 relationship types + owl:inverseOf axioms
  sources.ttl             # 32 secondary + 2 primary sources
  misc.ttl                # Sex, Status, DateType, NoteType
persons/
  CORN/                   # Cornelii (grouped by gens)
    CORN0017.ttl          # C. Cornelius (17) Cinna
    CORN0076.ttl          # P. Cornelius (76) Scipio
    ...
  IUNI/
    IUNI0001.ttl          # L. Iunius Brutus
    ...
concordances/             # Wikidata-derived cross-references
  wikidata.ttl            # 3,018 owl:sameAs links
  viaf.ttl                # 725 VIAF authority links
  nomisma.ttl             # 394 Nomisma numismatic concepts
  ...                     # + GND, LoC, British Museum, etc.
```

Files are named by DPRR ID (e.g., `IUNI0001`) and grouped into directories by the 4-letter nomen code. Each person file includes the person's triples plus all related assertions, notes, and references. Shared reference data (offices, sources, etc.) is not duplicated — person files contain URI references to entities in `reference/`.

## Changes from upstream

This dataset is derived from the [DPRR RDF dump](https://romanrepublic.ac.uk). All original URIs are preserved. The following changes were applied by [`shard.py`](shard.py):

### Stripped predicates

| Predicate | Reason |
|-----------|--------|
| `dprr:hasID` | Redundant — always equals the numeric suffix of the entity URI |
| `dprr:hasPosition` | UI display ordering for post assertions on the DPRR website |
| `dprr:hasOrderNumber` | UI display ordering for relationship types |
| `dprr:hasRelationshipNumber` | UI sorting for relationship display |
| `dprr:hasAssociatedWebpage` | Derivable from the entity URI (`http://www.romanrepublic.ac.uk/person/{id}`) |
| `dprr:hasDateDisplayText` | Defined in ontology but never populated (0 instances) |
| `dprr:hasEraDisplayText` | Never populated |
| `dprr:hasPatricianNotes` | Never populated |
| `dprr:isNobilisUncertain` | Never populated |
| `dprr:isNovusUncertain` | Never populated |
| `dprr:hasRelatedRelationship` | Never populated |
| `dprr:hasStatusAssertion` | Never populated |
| `dprr:hasAssertedRelationshipInverse` | Never populated |

### Stripped values

| Predicate | Values stripped | Reason |
|-----------|---------------|--------|
| `dprr:hasExtraInfo` | `"Autogenerated by add_inverse"`, `"Inferred Inverse"`, `"Inferred"` | Software provenance markers from DPRR's inverse-generation script. Other `hasExtraInfo` values (scholarly notes, source citations) are preserved. |

### Stripped types

| Type | Reason |
|------|--------|
| `dprr:RelationshipInverse` | Lookup table used by DPRR's `add_inverse` script to generate inverse relationship assertions. The inverse assertions are already materialized in the data. Zero incoming references — nothing points to these entities. The sex-conditional inverse logic (e.g., father→son if male, father→daughter if female) has been correctly applied. |

### Added triples

| What | Where | Reason |
|------|-------|--------|
| `owl:inverseOf` axioms | `reference/relationships.ttl` | 21 inverse-pair declarations derived from the stripped `RelationshipInverse` data. Expresses the same information (e.g., "father of" is inverse of "son of") using standard OWL, enabling reasoner-based inference. |

### Ontology changes

The ontology file preserves the upstream Protege-authored format. The following changes were applied:

- Removed definitions for all stripped predicates and the `RelationshipInverse` class
- Updated `rdfs:comment` on all major classes with clearer descriptions
- Renamed the default prefix from `:` to `dprr:` for clarity in cross-dataset contexts

### Orphan data

179 `PrimarySourceReference` entities in the upstream dump point to `PostAssertion` entities that no longer exist (0 triples). These dangling references are not included.

## Concordances

The `concordances/` directory contains cross-references to 12 external systems, all derived from [Wikidata](https://www.wikidata.org) using DPRR ID (property [P6863](https://www.wikidata.org/wiki/Property:P6863)) as the hub. Regenerate with:

```
python concordances.py .
```

## Prefixes

```turtle
@prefix dprr: <http://romanrepublic.ac.uk/rdf/ontology#> .
@prefix entity: <http://romanrepublic.ac.uk/rdf/entity/> .
```

## Deterministic output

Files are generated using [RDFC 1.0](https://www.w3.org/TR/rdf-canon/) canonicalization with sorted N-Quads to ensure byte-stable output. Running the sharding script twice on the same input produces identical files, so git diffs reflect only actual data changes.
